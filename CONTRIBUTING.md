\# Contributing to Points Are Bad



First off, thank you for considering contributing to Points Are Bad! It's people like you who make this game better for everyone.



\## How Can I Contribute?



\### Reporting Bugs

\- Check the \[Issues tab](https://github.com/frisbiz/points-are-bad/issues) to see if the bug has already been reported.

\- If not, open a new issue with a clear title, steps to reproduce, and expected vs. actual behavior.



\### Suggesting Enhancements

\- Open an issue with the tag `enhancement`.

\- Describe the feature and why it would be useful for the game.



\## Local Development Setup



1\. \*\*Fork the repository\*\* to your own GitHub account.

2\. \*\*Clone the fork\*\* to your local machine: git clone \[https://github.com/your-username/points-are-bad.git](https://github.com/your-username/points-are-bad.git)

3\. Install dependencies: npm install

4\. Set up environment variables:

Create a .env file based on .env.example.

Add your Firebase configuration keys.

5.Run the development server: npm run dev

The app should now be running at http://localhost:5173.





Style Guidelines

Branch Naming

Please use descriptive names for your branches:



feat/feature-name for new features.



fix/bug-name for bug fixes.



docs/changes for documentation.



Commit Messages

We follow basic conventional commits:



feat: ...



fix: ...



docs: ...



style: ... (UI changes, CSS)



Pull Request Process

Create a new branch for your changes.



Ensure your code follows the existing style of the project.



Update the README or documentation if necessary.



Submit the PR and wait for review!



Thank you for your contribution! ⚽️
